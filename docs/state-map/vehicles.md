# Vozidla (`vehicles`)

- **Soubory:** index.html (funkce vehicleCardHTML, vehicleRightHTML, bar, implementsHTML/implementsSummary, getFlashCls/getImplFlashCls) + bell.js (veh-fuel)
- **Počet stavů:** 29
- **biggestRisk:** Dvojí prahová nekonzistence kolem paliva + mrtvý fuel-flash kanál. (1) Lišta paliva (bar() index.html:305) přepíná na 'danger' už při pct<=20, ale zvonek (bell.js:18 FUEL_LOW=15) spustí alert až pod 15 %. V pásmu 15-20 % je tedy lišta jasně červená, ale ŽÁDNÝ alert se neobjeví — uživatel vidí kritickou červenou bez zvonku. (2) AdBlue: vehicleCardHTML renderuje has-adblue třídu i zlatý proužek z adBlueCapacity/adBluePercent (index.html:914, style.css:1238), ale zvonek hlídá výhradně fuelPercent — nízký AdBlue (scénář low-fuel má Fendt 942 adBlue 4 %, Fendt 1100 adBlue 3 %) nikdy nevyvolá alert, takže traktor s prázdným AdBlue tiše stojí. (3) getFlashCls('vehicles', v.name) je sice volán jako fallback ve flashCls (index.html:904), ale noteAllChanges (index.html:753) NEvolá žádný noteFlashRows/per-vehicle diff na fuelPercent — registruje se jen noteImplementFlashes. Fuel-flash větev řádku je tedy fakticky mrtvá a žádný test ji nepokrývá. Navíc #cnt-vehicles ukazuje vehicles.length (index.html:1130), nikoli počet viditelných řádků, takže s vehiclesActiveOnly=true počítadlo nesedí s DOM.
