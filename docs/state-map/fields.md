# Pole (`fields`)

- **Soubory:** index.html (field render — fruitName/prázdné, growthState/growthPercent, isReadyToHarvest, needsSowing, needsPlowing, needsLime, weedState, fertilizationLevel/sprayLevel, stoneLevel, daysToHarvest, owned/leased) + bell.js (field-ready)
- **Počet stavů:** 38
- **biggestRisk:** Dedikovaná st-fields-* sada (fields.spec.js) už pokrývá tag-priority (vč. isWithered/isCut přes scenarioFieldsTagPriority), growth-bar prahy, days-to-harvest, condition badge weedState 2/3/4/5, action chips, spray/extra sloupce, crop/farmland buňku, bell-ready i empty-state. Zbylé reálné mezery: (1) chip 'Válet' — needsRolling není v žádné plodině AVAIL_FRUITS, takže chip nikdy nesvítí (test.fixme, blocked); (2) 'Všechna pole jsou skrytá' — vyžaduje seed TableTools hidden setu, filtr-cesta jde na 'Žádná pole' (test.fixme, blocked); (3) crop-cell fallback — scénář používá SUGARBEET, který MÁ ICON_MAP záznam, takže vykreslí .crop-ic místo .crop-ic-fallback → potřebuje scenario-fix (mimo-katalog plodina bez ICON_MAP). POZN: weedState (0-9, badge) ≠ weedLevel (0-3, chip+sloupec) — dvě různá pole pro 'plevel', nepleť je při změně savegame.js parseru.

> Povrch: Pole (Fields) — sec-fields table on index.html (renderFields + fieldStateTag + fieldNeedsChips + fieldConditionBadge + GROUND_TYPE_LABELS + bar) and the field-ready bell alert in bell.js
