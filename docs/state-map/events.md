# Události (`events`)

- **Soubory:** index.html (events feed — typy sowing/harvest/spray, dedup klíč timestamp\|fieldId\|type, timestamp)
- **Počet stavů:** 32
- **biggestRisk:** Dedup klíč flash mechanismu: flashes Map je keyed ${section}::${rowKey}, ale rowKey pro storage+productions přidává header:: a item:: prefix. Pokud byly stavy mergovány špatně nebo _flashChannelOf() logika je buggy, duplikáty z po sobě jdoucích ticků mohou být trackovány jako NOVÝ flash místo toho, aby se ignorovaly. Specificky: noteImplementFlashes() trackuje per vehicles::impl::${vName}::${implName}::${i} — jakmile fillUnit setiny jednu litr za tick (500l nádrž → 0.2 % per tick), flash by měl EXPIRE po 10s, ale pokud expiry timer není zavolán nebo memo se nezadíšně, flash zůstane pinned trvale. Testovací scénář: vozidlo plní vlečku 1l/tick po dobu 30s. Očekávaný výsledek: flash se zapne, trvá 10s, pak zhasne, a nový flash se může spustit až po expiry. Riziko: bez zavolání scheduleFlashCleanup() nebo jiného mechanismu, můžou být živé flashes neočekávaně perzistentní, zejména když WS ticky do na dva sekundy skončí.

> Povrch: Dashboard (index.html) — Živý povrch s 6 sekcemi: Pole, Vozidla, Zvířata, Sklady, Výrobny, Ceny + KPI řada s 4 kartami
