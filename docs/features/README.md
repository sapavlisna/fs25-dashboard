# Features — backlog & specs

Složka pro **rozepsané featury** Dashboardu — než se začne kódit, sem přijde
podrobný plán (cíl, současný stav, návrh, kroky, rizika). Po dokončení se ve
specu doplní status `HOTOVO` + odkaz na commit, ať je dohledatelné, co a proč
se dělalo.

Konvence:

- Jeden soubor = jedna featura. Název `YYYY-MM-DD-kebab-nazev.md`.
- Status v hlavičce: `NÁVRH` → `ROZPRACOVÁNO` → `HOTOVO` (+ commit hash).
- Otevřené otázky drž v sekci „Rozhodovací body" nahoře, ať se na ně nezapomene.
