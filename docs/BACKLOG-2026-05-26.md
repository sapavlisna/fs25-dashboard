## Bod 1 — Vehicles: flash změn nereaguje na změnu naplnění vlečky [BUG]

### Problém
Když je připojená vlečka a mění se její naplněnost (přibývá/ubývá plodina), nepropíše se
to do "flash změn" (zelený/červený záblesk řádku). Dnes flash naskočí jen při překročení
hranice 0 % (prázdno) nebo 100 % (plno) — `noteImplementBoundaryFlashes()` v
`index.html`. Mezilehlé změny (např. 30 % → 45 %) se ignorují.

### Analýza (kód)
`index.html:noteImplementBoundaryFlashes` flashne jen `reachedFull || reachedEmpty`.
Sklady přitom flashují na JAKOUKOLIV změnu (`noteFlashes` up/down). Vlečky by se měly
chovat jako sklady — každá změna fill levelu = flash.

### Plán
- Rozšířit `noteImplementBoundaryFlashes` (přejmenovat na `noteImplementFlashes`):
  flash up/down při jakékoli změně `percent` daného fillUnitu, ne jen na hranicích.
- Zachovat pojistku proti "pinned" flashům (nový flash jen když není aktivní — stejně
  jako sklady).
- `getImplFlashCls` zůstává (čte agregovaný flash pro řádek vozidla).

### Testy (desktop + mobil)
- `vehicles-extra.spec.js`: mock scénář, kde vlečka mění fill 20 % → 40 % mezi ticky →
  assert řádek vozidla dostane `flash-up`; pokles → `flash-down`.
- Ověřit na 1440px i 375px (flash class je nezávislá na layoutu, ale ověřit oboje).

### Akceptační kritéria
- [ ] Připojená vlečka se plní → řádek vozidla blikne zeleně i při mezilehlé změně.
- [ ] Vyprazdňuje se → blikne červeně.
- [ ] Flash respektuje per-sekci toggle (vypnutý flash vozidel = nic).
- [ ] Žádný "pinned green" při pomalém kontinuálním plnění (1 flash, ne každý tick).

## Bod 2 — Vehicles: naplněnost vlečky ukazovat vždy, ne jen když > 0 [ENHANCEMENT]

### Problém
Prázdná připojená vlečka (0 %) se v základním pohledu nezobrazí vůbec — summary
(`implementsSummary`) skrývá prázdné fillUnity, dokud není zapnutý setting
`vehicleShowEmptyImplements`. Uživatel chce vidět připojené nářadí s naplněností
**vždy** od okamžiku připojení, i na 0 %.

### Analýza (kód)
`implementsSummary` (basic view): `if (isEmpty && !showEmpty) continue;` → prázdné
skryté. `implementsHTML` (expanded view) prázdné UŽ ukazuje jako "—". Fix se týká
hlavně basic-view summary.

### Plán
- Změnit default `vehicleShowEmptyImplements` na `true` NEBO v summary vždy zobrazit
  připojené nářadí s fillUnit (0 % jako tlumené "—"). Volím: **vždy zobrazit** v summary
  (prázdné tlumeně), setting ponechat jako "skrýt prázdné" override (invertovat smysl
  nebo nechat jako doplněk). Konzistence s expanded view, kde se prázdné ukazuje taky.
- Vizuál prázdné: `c-muted`, label "—", `0 %`.

### Testy (desktop + mobil)
- `vehicles-extra.spec.js`: mock vozidlo s prázdnou vlečkou (0 %) → assert summary
  obsahuje řádek nářadí (ne skrytý).
- Mobil 375px: ověřit, že přidaný řádek nepřeteče kartu vozidla.

### Akceptační kritéria
- [ ] Připojená prázdná vlečka je vidět hned (0 %), bez nutnosti měnit nastavení.
- [ ] Při plnění se číslo aktualizuje (a blikne — viz bod 1).
- [ ] Nářadí bez fillUnitu (pluh) se dál nezobrazuje (filtr `capacity > 100 L` v Lua).

## Bod 3 — Výrobny: ukazuje jen jednu (sýr), chybí ostatní [BUG, velký]

### Problém
Stránka Výrobny zobrazuje jen jednu výrobnu, ač jich má hráč na mapě víc.

### Findings (fs25-api-researcher)
- Současný `collectProductions()` iteruje `g_currentMission.placeableSystem.placeables`
  + `isOwnedBy()` + `spec_productionPoint`. **Root cause:** `isOwnedBy()` /
  `pp:getOwnerFarmId()` selže pro výrobny s `isFinalized = false` — `onFinalizePlacement`
  přepíše owner na `AccessHandler.EVERYONE` (=0), takže ownership check je zahodí.
  Navíc `ownerId > 0` v `isOwnedBy` zahodí neutrální výrobny.
- **Správný zdroj:** `g_currentMission.productionChainManager:getProductionPointsForFarmId(farmId)`
  → vrací přímo `ProductionPoint` objekty patřící farmě, bez ruční iterace placeables.
  `ProductionChainManager` = `giantsDocu/…category_57_class_560.md`.
- `pp.storage` je u validní výrobny vždy non-nil; `pp.owningPlaceable` → zpět na placeable
  pro název.

### Plán
- Přepsat `collectProductions()` enumeraci na
  `g_currentMission.productionChainManager:getProductionPointsForFarmId(farmId)`.
- Fallback: pokud `productionChainManager` nil (starší verze / edge), ponechat starou
  iteraci placeables jako záložní cestu.
- Zbytek (storage fills, recipes loop) beze změny.
- `[DIAG]` log: počet nalezených výroben, ať na živém testu vidíme, že chytíme všechny.

### Testy
- **Mock** `productions.spec.js`: scénář s ≥ 3 výrobnami (mlékárna, pekárna, pila) →
  assert frontend vykreslí všechny 3 (ne jednu). Mock posílá `productions[]` jak by je
  vrátil opravený kód.
- **Živě** (`/fs25-test --slot=3`): postavit/mít víc výroben → log `[DIAG]` ukáže count;
  dashboard zobrazí všechny. Hru zabít bez uložení.

### Akceptační kritéria
- [ ] Všechny hráčovy výrobny se zobrazí (ne jen jedna).
- [ ] Funguje i pro "neutrální"/nedokončené výrobny vlastněné farmou.
- [ ] Graceful fallback když `productionChainManager` chybí.
- [ ] `knowledge/fs25-production-points.md` zapsán.

## Bod 4 — Vehicles: minimalisticky poškození + rychlost [UX + research]

### Findings (fs25-api-researcher)
- **Damage (kondice):** `vehicle:getDamageAmount()` → 0..1. **Kondice v menu = `(1-damage)*100 %`.**
  Guard: `if v.spec_wearable and v.getDamageAmount`. (`…category_77_class_829.md`)
- **Rychlost:** `vehicle:getLastSpeed()` → **už v km/h** (interně m/ms × 3600). Bez konverze.
  Parkované ≈ 0. (`…category_90_class_886.md:1985`)
- **Max rychlost:** `vehicle.spec_motorized.motor:getMaximumForwardSpeed()` → **m/s**, ×3.6 = km/h.
  Jen motorized.

### Plán
- **Lua** `collectVehicles()`: přidat `rec.conditionPercent = floor((1-getDamageAmount())*100)`
  (guarded) a `rec.speedKmh = floor(getLastSpeed()+0.5)` (jen když isInUse / speed > 0).
  Max speed volitelně.
- **Frontend** `vehicleCardHTML`: minimalistický indikátor — kondice jako malý % badge
  (zelená ≥ 80, žlutá ≥ 50, červená < 50); rychlost km/h jen když vozidlo jede.
- **Setting** v ⚙ → Vozidla: toggle "Zobrazit kondici + rychlost" (default zap nebo vyp —
  volím default zapnuto pro kondici, rychlost jen při jízdě). DashState klíč
  `vehicleShowCondition` (sync).

### Testy (desktop + mobil)
- `vehicles-extra.spec.js`: mock vozidlo s `conditionPercent: 45` → assert červený badge;
  `speedKmh: 18` + isInUse → assert "18 km/h" visible; stojící (speed 0) → rychlost skrytá.
- Mobil 375px: badge se vejde do karty, nepřetéká.

### Akceptační kritéria
- [ ] Kondice vozidla viditelná jako barevný % (zelená/žlutá/červená).
- [ ] Rychlost km/h jen u jedoucího vozidla.
- [ ] Toggle v nastavení funguje + syncuje.
- [ ] `knowledge/vehicle-damage-speed-api.md` zapsán.
