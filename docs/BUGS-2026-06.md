# Dashboard Bugs – June 2026

Found during smoke test coverage run (state/ spec suite).

## BUG-01: History mock endpoints missing for balance/price history injection — RESOLVED

**Symptom:** History.spec.js tests `st-history-balance-*` and `st-history-price-*` all fail because
`POST /mock/balance-rows` and `POST /mock/price-rows` endpoints do not exist in index.js.
These tests require server-side injection of test history rows into the JSONL store.

**Fix (done):** Added `POST /mock/seed-history` (+ `/mock/balance-rows`, `/mock/price-rows` aliases)
in `index.js`, gated behind `DASHBOARD_MOCK=1`. Backed by `db.seedHistory()` which rewrites
`data/balance.jsonl` / `data/prices.jsonl` with the supplied rows and freezes the live watcher off
the current game-day so a seeded row isn't clobbered. Also hardened `appendJsonl`/`writeJsonl` to
`mkdirSync` the data dir (the sandbox `data/` dir didn't exist at write time → live history was never
being written in the smoke env — a latent bug this fix also closes).

## BUG-02: Vehicles custom-payload scenario not supported

**Symptom:** Several vehicles.spec.js tests used `POST /mock/scenario` with an inline `payload` field.
The server ignores `payload`. (Superseded — the affected tests were reworked to use named scenarios,
which now all exist.)

## BUG-03: .v-implements always hidden — CSS missing display for expanded vehicles — RESOLVED

**Symptom:** `style.css` set `.expanded-vehicles .v-implements { display: none; }`, hiding the full
per-fillUnit stack on desktop entirely. A 2026 expanded-layout redesign regressed this from the
original `display: flex` (commit f216127), where the expanded vehicle view was meant to show the
full stack.

**Fix (done):** Restored `.expanded-vehicles .v-implements { display: flex; }`. The vi-row smoke test
now runs with `vehiclesExpanded=true` and asserts the per-implement bars/colours.

## BUG-04: forecastChart not on window — Canvas chart tests cannot inspect Chart.js data — RESOLVED

**Symptom:** `history.html` kept `forecastChart` (and `balanceChart` / `priceChart`) as module-level
variables, so `page.evaluate(() => window.forecastChart)` always returned null.

**Fix (done):** `history.html` now assigns `window.forecastChart` / `window.balanceChart` /
`window.priceChart` after each chart build, plus `window.__forecastRenderCount` (memo-hit detection).
`app.js` exposes `window.__ws` so tests can drive a WS disconnect. The forecast watch/current-month/
memo-hit and balance pointRadius / price legend tests now assert against the real Chart.js instances.

## Remaining consciously-blocked smoke states (left as test.fixme)

- **Events feed** — no frontend exists (server only stores `/api/events`); nothing to assert.
- **Bell badge "9+"** — `genAlerts` emits at most 7 keys, so n>9 is unreachable dead code.
- **kpi balance flash-up/down (boundary-ok)** — needs two tightly-synced WS ticks across the 5 s mock;
  use the dedicated flash harness (`playwright.flash.config.js`, port 3098) instead.
- **calendar ws-connecting / waiting-for-data** — transient pre-first-payload state; the mock emits
  immediately, so it can't be captured without a server-side WS pause.
- **calendar bar-cut / needs-roller, dashboard rollerLevel "Válet" chip, fields rollerLevel** — require a
  fruit catalog carrying `needsRolling:true` (none does) and the calendar fruitMeta is keyed by fruit
  NAME while the chip looks up by fruitTypeId — keying ambiguity makes a faked scenario unreliable.
- **themes "field detail modal cond badges"** — misframed: the dashboard has no field-detail modal;
  `.cond-badge` only exists in calendar.html. Belongs in calendar.spec.js.
